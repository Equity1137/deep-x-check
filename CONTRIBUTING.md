# Contributing to DeepXCheck

We love your input! We want to make contributing to DeepXCheck as easy and transparent as possible.

## How to Contribute

### 1. Reporting Bugs
- Use the GitHub Issues tracker
- Describe the bug in detail
- Include steps to reproduce
- Add screenshots if applicable

### 2. Suggesting Enhancements
- Open an Issue with the "enhancement" label
- Explain why this enhancement would be useful
- Suggest implementation if possible

### 3. Adding New Detection Patterns
- Fork the repository
- Add your pattern detection method in `deepxcheck.py`
- Update documentation if needed
- Submit a Pull Request

### 4. Improving Documentation
- Fix typos, clarify explanations
- Add examples or tutorials
- Translate into other languages

## Development Process

1. **Fork** the repo on GitHub
2. **Clone** your fork locally
3. **Create a branch** for your changes
4. **Commit** your changes
5. **Push** to your fork
6. Submit a **Pull Request**

## Code Style
- Follow PEP 8 for Python code
- Add comments for complex logic
- Write meaningful commit messages

## Questions?
Open an Issue or start a Discussion. We're here to help!

Thank you for contributing to a more transparent digital world!
